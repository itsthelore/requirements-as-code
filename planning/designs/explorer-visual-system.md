# Explorer Visual System

## Context

RAC Explorer should feel like a modern developer tool while remaining terminal native.

## User Need

Users need a calm, information-dense interface for understanding complex repositories.

## Design

### Surfaces

Three depth levels:

Surface 0:
Application background

Surface 1:
Panels

Surface 2:
Focused elements


### Colour

Single accent colour.

Colour communicates focus, not meaning.


### Layout

Stable regions:

Navigation

Context

Status


### Interaction

Primary:

/

Secondary:

keyboard shortcuts


## Constraints

- Terminal compatibility
- 256 colour fallback
- No colour-only meaning
- Keyboard first

## Accessibility

Validation state uses:

icons + labels + colour
