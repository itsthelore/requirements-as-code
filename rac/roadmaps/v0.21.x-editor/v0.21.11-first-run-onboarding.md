# RAC v0.21.11 — First-Run Onboarding & rac Provisioning

## Status

Achieved

## Context

The thin client is inert without a resolvable `rac`, and most installs that hit
a missing-`rac` cliff churn before ever seeing the product work. A new user's
first session is make-or-break; today a missing or version-skewed `rac` yields a
single warning and a dead extension. After publication (v0.21.10), getting every
install to the enforcement "aha" — the moment it catches a reference to a
ruled-out decision — is the highest-leverage adoption lever.

## Outcomes

- An install detects a missing or skewed `rac` and guides the user to a working
  setup rather than failing silently.
- A fresh workspace can be scaffolded into a valid RAC corpus.
- The first session reaches the enforcement "aha" on rails.

## Initiatives

### Initiative 1 — Detect & guide

On activation, verify `rac` presence and version (reusing the existing skew
check); on a miss, offer a guided install (pipx/uv) and docs — driven by a
deterministic `rac --version` / capability probe rather than parsing stderr.

### Initiative 2 — Scaffold a workspace

A command that scaffolds `.rac/config.yaml` and a starter corpus folder, so an
empty repository becomes a valid corpus the rest of the extension can act on.

### Initiative 3 — Get-Started walkthrough

A VS Code walkthrough that ends by deliberately staging the enforcement
diagnostic, so the user sees a caught contradiction within the first session.

## Constraints

- Thin client (ADR-063); offline-capable guidance.
- No telemetry — the on-ramp is designed without measurement.
- No install without explicit consent.

## Non-Goals

- Bundling or vendoring the `rac` binary.
- Auto-installing engines silently.

## Implementation Contract

- A missing/skewed `rac` produces actionable guided setup, not just a warning.
- A scaffold command creates a valid `.rac/config.yaml` and starter folder.
- A walkthrough stages a live enforcement diagnostic in the first session.

## Success Measures

- With `rac` absent, a new user is guided to a working install and a scaffolded
  corpus.
- The walkthrough surfaces a real enforcement diagnostic on first run.

## Risks

- Install flows vary across OS/Python setups. Mitigated by guidance and links
  over silent auto-install.
- A deterministic capability probe may need a small additive CLI surface.

## Assumptions

- A deterministic `rac` version/capability probe is available or addable
  additively (ADR-007).
