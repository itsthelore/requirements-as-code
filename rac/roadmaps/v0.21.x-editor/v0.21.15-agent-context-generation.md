# RAC v0.21.15 â€” Agent Context & Rules Generation

## Status

Planned

## Context

The editor enforces against the human, but the AI agent that re-litigates settled
decisions never sees the corpus. Per ADR-067, the wedge is committed, drift-
guarded agent-rules files generated by `rac` from the live corpus â€” they reach
every agent (including Copilot) with zero per-developer setup and are reviewable
in a PR, where an MCP-only design reaches only power users. This release builds
that generation, the one-command editor setup, and the CI drift gate.

## Outcomes

- `rac` emits per-client agent-rules files (`CLAUDE.md`, `AGENTS.md`,
  `.cursor/rules`, `.github/copilot-instructions.md`) deterministically from the
  live corpus.
- A single editor command sets up agent integration â€” generates the files and
  registers the `lore` MCP server per client.
- A CI `--check` gate keeps the generated projection from silently rotting.

## Initiatives

### Initiative 1 â€” `rac export --agent-rules [--check]`

A new additive CLI verb that emits a distilled projection of live (Accepted,
non-superseded) decisions and scope fences into a fenced managed block per client
format, with a provenance hash; `--check` exits non-zero when a committed file
has drifted from the corpus (the vendored-shell drift discipline).

### Initiative 2 â€” "RAC: Set up agent integration"

An editor command that detects present agents, generates the rules files, and
registers `lore` per client (`registerMcpServerDefinitionProvider` for VS Code;
a generated `mcp.json` for Cursor / Claude Code â€” consent stays the user's), then
watches `rac/**` and offers regeneration on change.

### Initiative 3 â€” First-run nudge

The setup ends by pointing the user at the first "ask your agent: what did we
decide about X?" moment, so value lands in the first session.

## Constraints

- Thin client (ADR-063): generation and all corpus logic live in `rac`; the
  extension registers, watches, and presents â€” it computes nothing.
- Additive (ADR-007); deterministic, offline, no telemetry.
- The block is distilled (bans + closed-decision pointers), not the whole corpus,
  and is drift-guarded (ADR-067).

## Non-Goals

- A semantic verdict in the engine (ADR-067).
- Inlining the whole corpus into the rules files.
- Auto-enabling MCP servers without user consent.

## Implementation Contract

- `rac export --agent-rules` emits drift-guarded per-client files; `--check` exits
  non-zero on drift.
- The editor command generates the files and registers `lore` per client.
- A corpus change prompts regeneration.

## Success Measures

- A generated managed block lists the live bans and closed decisions for each
  target format.
- Editing the corpus then running `--check` fails until the files are regenerated.
- After setup, an MCP-capable agent can query the corpus via `lore`.

## Risks

- Generated files drifting from the corpus. Mitigated by the provenance hash and
  `--check` gate.
- Per-client format churn and attention-budget bloat. Mitigated by distillation
  and a small managed block.

## Assumptions

- `rac` can deterministically project live artifacts into a stable digest.
- VS Code's `registerMcpServerDefinitionProvider` and per-client config files are
  available; consent remains the user's.
