# RAC v0.21.18 — Safe Rename & Refactor

## Status

Planned

## Context

Renaming an artifact id by hand across the corpus is the sharpest authoring fear —
a botched rename silently corrupts cross-artifact links, and structural validation
may not even catch a rename that is internally consistent but wrong. This release
makes rename safe through an additive engine contract that computes and applies
the corpus-wide edit set, surfaced in the editor. It is independent of the
agent-enforcement work but completes the editor's authoring story (the design
council's item E), CLI-gated so the extension never computes the edit set
(ADR-063) and lands on the extension test harness.

## Outcomes

- Renaming an artifact id (or alias) updates every inbound reference atomically.
- An "add relationship" code action inserts a resolvable reference.
- Quick-fix bodies are schema-guided rather than TypeScript-templated.

## Initiatives

### Initiative 1 — Engine refactor contract

An additive `rac rename --json` that computes and applies the corpus-wide edit set
deterministically and reversibly; the extension never computes references.

### Initiative 2 — Editor rename

A rename command/refactor over the contract that previews the edit set before
applying it.

### Initiative 3 — Authoring actions

An "add relationship" code action, and quick-fix bodies sourced from `rac schema`
(closing the additive-contract drift risk in the current TypeScript-templated
missing-section quick-fix).

## Constraints

- Thin client (ADR-063): the edit set is computed by `rac`, never the extension.
- Additive (ADR-007); deterministic and reversible.

## Non-Goals

- Editor-side reference computation.
- Semantic refactors or renaming files outside the corpus.

## Implementation Contract

- `rac rename --json` computes and applies a deterministic corpus-wide edit set.
- The extension previews and invokes it.
- Quick-fix bodies come from `rac schema`, not TypeScript templates.

## Success Measures

- Renaming a referenced artifact id updates every inbound reference and leaves
  `rac relationships --validate` clean.
- An add-relationship action inserts a resolvable reference.
- Quick-fix bodies match the canonical schema.

## Risks

- A write-path contract is new engine surface. Mitigated by an additive,
  deterministic, reversible design plus fixture tests.
- A botched rename corrupts links. Mitigated by the engine-owned edit set and the
  extension test harness.

## Assumptions

- A deterministic, reversible rename/refactor contract is addable to `rac`.
- `rac schema` can drive quick-fix bodies.
