# RAC v0.21.17 — Claude Code Pre-Edit Hook

## Status

Planned

## Context

ADR-067 establishes that there is no cross-platform pre-edit veto on agent
suggestions — with one exception: Claude Code's `PreToolUse` hook can block an
Edit/Write before it lands. The extension *generates* that hook (it cannot itself
be the interceptor), having it pipe proposed content to `rac validate -`. This is
the single highest-fidelity enforcement seam — client-specific and opt-in — while
other clients rely on the post-edit guard from v0.21.16.

## Outcomes

- An opt-in, generated Claude Code `PreToolUse` hook validates proposed edits and
  blocks structural contradictions before they land.
- The honest boundary is preserved: no claim of cross-platform interception.

## Initiatives

### Initiative 1 — Generated hook

The extension emits a `.claude/` `PreToolUse` hook configuration that pipes the
proposed content to `rac validate -` and blocks on a structural finding. The
stdin validation path largely exists already.

### Initiative 2 — Opt-in and scoping

The hook is opt-in, clearly labelled as client-specific, and scoped to structural
contradictions (references to retired or missing decisions), per ADR-067.

## Constraints

- Thin client (ADR-063): the hook calls `rac`; it computes nothing.
- Structural enforcement only; deterministic and offline.
- Honest scope — no claim of intercepting Copilot or Cursor edits.

## Non-Goals

- Intercepting Copilot inline or Cursor agent edits (no platform API).
- Semantic verdicts.

## Implementation Contract

- A generated, opt-in Claude Code `PreToolUse` hook validates proposed content via
  `rac validate -` and blocks structural contradictions.
- Other clients rely on the v0.21.16 post-edit guard.

## Success Measures

- With the hook enabled, a Claude Code edit introducing a reference to a retired
  decision is blocked pre-write with the finding.
- Disabling the hook falls back cleanly to post-edit diagnostics.

## Risks

- Claude Code hook-config format churn. Mitigated by generating the hook from one
  place.
- Over-blocking. Scoped to structural findings; the stdin path already exists, so
  implementation risk is low.

## Assumptions

- Claude Code `PreToolUse` hooks can run a command and block on its exit code.
- `rac validate -` accepts proposed content on stdin (it already does).
