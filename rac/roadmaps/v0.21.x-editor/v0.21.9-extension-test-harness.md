# RAC v0.21.9 — Extension & Viewer Test Harness

## Status

Achieved

## Context

The extension and the `lore-web` viewer have no behavioural tests — only `tsc`
and a successful bundle, which prove the code is type-correct and exists, not
that it works. v0.21.8 shipped a graph-layout bug (the graph rendered at ~15%
scale and looked broken) that nothing in CI would have caught; it surfaced only
after release via ad-hoc jsdom mounting. As the extension heads for publication
(v0.21.10), an untested client is a regression amplifier against the product's
own thesis — under "enforcement is the product" (ADR-049), a false-negative
diagnostic is the product silently lying. This release builds the smoke-and-
contract test gate that everything after it lands on.

## Outcomes

- A CI gate mounts the viewer and exercises extension providers against a
  fixture corpus.
- The graph-layout regression class (the v0.21.8 bug) is caught automatically.
- Publication (v0.21.10) can be gated on a green test run.

## Initiatives

### Initiative 1 — Viewer smoke & determinism tests

A jsdom mount of the viewer `App` at each route, plus a seeded-graph determinism
snapshot asserting node bounds and fit scale — the exact assertion that would
have failed on the v0.21.8 ~15%-scale bug before release.

### Initiative 2 — Extension provider tests

`@vscode/test-electron` tests over a deterministic fixture corpus: open a corpus
and assert diagnostics appear, a relationship violation surfaces at the
reference site, and completion / hover / quick-fix smoke-pass.

### Initiative 3 — CI wiring as the pre-publish gate

Run both on PRs and `main`, and make a green run a prerequisite for the
v0.21.10 publish pipeline.

## Constraints

- Thin client (ADR-063): tests assert against `rac --json` contracts and the
  rendered DOM, never re-deriving engine logic.
- Smoke-and-contract scope, not exhaustive coverage — raise the floor without a
  velocity tax.
- Deterministic fixtures; no network.

## Non-Goals

- Full end-to-end coverage or visual pixel-diffing.
- Testing `rac` itself — the engine has its own suite and drift-guard.

## Implementation Contract

- A jsdom viewer test (including the seeded-graph determinism snapshot) and a
  `@vscode/test-electron` provider test over a fixture corpus, both runnable in
  CI.
- A green run of both is a declared prerequisite for publishing (v0.21.10).

## Success Measures

- The v0.21.8-class layout regression is reproduced by a failing test prior to
  its fix.
- Opening a fixture corpus asserts diagnostics and a relationship finding.
- CI fails on a seeded-graph scale/transform regression.

## Risks

- Over-indexing on coverage slows shipping. Mitigated by smoke-and-contract
  scope.
- jsdom is not a real browser. Mitigated by asserting logic/coordinates rather
  than pixels, with `@vscode/test-electron` covering the real host.

## Assumptions

- jsdom can mount the viewer (proven during the v0.21.8 fix) and
  `@vscode/test-electron` runs in CI.
