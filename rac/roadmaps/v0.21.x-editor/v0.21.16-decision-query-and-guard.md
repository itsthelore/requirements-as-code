# RAC v0.21.16 — Decision Query & Post-Edit Guard

## Status

Planned

## Context

With context supplied (v0.21.15), an agent can consult the corpus; this release
adds the live decision query — the "what did we decide about X?" surface — and the
honest post-edit guard that catches contradictions the rules files do not prevent.
The existing structural diagnostics already fire on agent-written files; this adds
a save-time backstop, a reference-site quick-fix, and a CI rules-staleness gate.
Per ADR-067 the engine retrieves live decisions deterministically and the agent
judges contradiction — no semantic verdict ships in the engine.

## Outcomes

- A one-keystroke decision query, available as an MCP tool, a command-palette
  action, and a CLI face.
- A save-time contradiction backstop and a reference-site quick-fix that work
  regardless of which agent wrote the change.
- Rules staleness enforced in CI.

## Initiatives

### Initiative 1 — `find_decisions` and its surfaces

An additive `lore` tool returning ranked live decisions for a topic (search
filtered to `type=decision`, with a description engineered to fire on "what did we
decide / is X ruled out"), plus a command-palette action and a CLI face over
`rac find`.

### Initiative 2 — Post-edit guard

An `onWillSaveTextDocument` save-gate and a reference-site quick-fix layered over
the existing diagnostics, surfacing a structural contradiction before the bad
state solidifies — client-agnostic, since a save is a save.

### Initiative 3 — Drift gate in CI

Wire `rac export --agent-rules --check` into CI so the generated rules files
cannot silently rot relative to the corpus.

## Constraints

- Thin client (ADR-063); the query is a query, not a guardrail.
- Structural enforcement only (ADR-067) — no semantic scoring.
- Additive (ADR-007); deterministic, offline.

## Non-Goals

- Semantic contradiction scoring.
- Pre-edit interception of agent edits (no platform API; ADR-067).

## Implementation Contract

- `find_decisions` returns ranked live decisions for a topic across MCP, palette,
  and CLI.
- A save-gate surfaces a structural contradiction before save, with a
  reference-site quick-fix.
- `--agent-rules --check` gates CI on rules freshness.

## Success Measures

- Asking "what did we decide about X" returns the relevant decisions in the
  palette, to the agent, and on the CLI.
- Saving a corpus file that references a retired decision is gated, with a
  quick-fix offered.
- A stale rules file fails CI.

## Risks

- Query false relevance. Mitigated by deterministic retrieval with no semantic
  scoring.
- Save-gate friction. Scoped to structural findings only.

## Assumptions

- Search filtered to decisions is sufficient for the query.
- The existing diagnostics fire on agent-written files (confirmed).
