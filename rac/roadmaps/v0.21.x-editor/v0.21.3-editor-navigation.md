# RAC v0.21.3 — Editor Navigation

## Status

Planned

## Context

v0.21.0 gave the extension hover and go-to-definition on artifact IDs. A corpus
is a graph, and moving through it is a core authoring activity: which artifacts
point here, what does this reference actually say, where is every artifact. This
release completes editor navigation over the corpus, each feature backed by an
existing `rac` lookup (ADR-063).

## Outcomes

- Hover answers "is this still current?", not just "what is this" — it shows the
  target's lifecycle status and a snippet.
- A maintainer can see every artifact that references the one they are editing.
- Artifact references are clickable, the document outline shows its sections, and
  any artifact is reachable by ID/title from anywhere.

## Initiatives

### Initiative 1 — Hover enrichment

Extend the v0.21.0 hover to include the target's lifecycle status (e.g.
⚠ Superseded, ADR-051) and a short body snippet (from `rac export` /
`rac resolve`), so hover doubles as a currency check.

### Initiative 2 — Find-all-references

A reference provider returning incoming links — every artifact that references
the one under the cursor — computed from the export's resolved `from → to` edges
(`rac export` carries them keyed by id), anchored at the referencing token in
each source. No extra `rac` call beyond the cached export.

### Initiative 3 — Links, outline, and workspace symbols

Document links (clickable aliases, resolved against the export's alias index), a
document-symbol provider exposing the artifact's `##` headings in the Outline
view (pure Markdown), and a workspace-symbol provider listing every artifact by
title (from the cached export).

## Constraints

- Thin client only (ADR-063): hover status/snippet, incoming edges, alias→file
  links, and the symbol set all come from the cached `rac export`; the extension
  re-derives no relationships.
- Symbol and reference lookups are served from the per-folder export cache
  (introduced in v0.21.2, hardened in v0.21.6) to stay responsive.

## Non-Goals

- A graphical relationship view (v0.21.5).
- Rename/refactor of artifact IDs across the corpus.
- Outline entries beyond the artifact's own sections.

## Implementation Contract

- Hover shows title, type, ID, path, lifecycle status, and a body snippet.
- Find-all-references returns incoming-reference locations.
- Document links, outline (document symbols), and workspace symbols resolve to
  artifact files.

## Success Measures

- Hovering a reference to a retired artifact shows its retired status.
- Find-all-references on an artifact lists the artifacts that reference it.
- Ctrl/Cmd-clicking an ID opens the target; the Outline lists the sections;
  workspace-symbol search jumps to an artifact by title.

## Risks

- Repeated lookups (hover, symbols) spawning `rac`. Mitigated by the cached index
  (v0.21.6).
- Incoming-reference computation cost on large corpora. Mitigated by caching and
  on-demand resolution.

## Assumptions

- `rac relationships` / `get_related`, `rac export`/`rac resolve`, and
  `rac find`/index expose incoming links, status, snippets, and a symbol set.
