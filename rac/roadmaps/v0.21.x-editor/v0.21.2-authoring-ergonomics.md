# RAC v0.21.2 — Authoring Ergonomics

## Status

Planned

## Context

With validation and enforcement in the editor (v0.21.0–v0.21.1), the next
friction is authoring: a maintainer hand-types artifact IDs into relationship
sections, hand-builds section skeletons, and leaves the editor to scaffold a new
artifact. Each is a place the extension can help, and each is backed by an
existing `rac` capability (`find`, `improve`, `schema`, `new`), so it stays a
thin client (ADR-063).

## Outcomes

- Typing in a relationship section offers artifact-ID completions; accepting one
  inserts a reference that resolves.
- A structural finding (a missing recommended section) can be fixed with a single
  code action.
- A new artifact can be scaffolded from inside the editor, of any type.

## Initiatives

### Initiative 1 — Artifact-ID completion

A completion provider that, inside a relationship section (`Related Decisions`,
`Related Roadmaps`, …), offers artifact **aliases** (the human ones, e.g.
`adr-007`, not the opaque id) drawn from the corpus export (`rac export`, cached
per folder and invalidated on save); VS Code filters by what is typed. Accepting
an item inserts a resolvable reference.

### Initiative 2 — Quick-fix code actions

Code actions for structural `missing-<section>` diagnostics — e.g. "Insert
'## Success Metrics' section" — that append the canonical section heading so the
finding clears. The section name is taken from the diagnostic code; richer
guidance bodies from `rac schema` / `rac improve --template` are a follow-on.

### Initiative 3 — New-artifact scaffolding

A "RAC: New Artifact" command that picks a type and scaffolds via `rac new`
(opening the result), plus per-type section snippets for in-place authoring.

## Constraints

- Thin client only (ADR-063): completions from `rac find`/index, skeletons from
  `rac improve`/`rac schema`, scaffolding from `rac new`. No template logic is
  reimplemented in the extension.
- Completions and quick-fixes are served from the cache (v0.21.6) where possible
  to stay responsive.

## Non-Goals

- A full authoring framework or WYSIWYG editor (RAC is not a content store).
- Auto-rewriting artifact bodies beyond inserting canonical section skeletons.
- Completion for free-text prose; this targets artifact references and sections.

## Implementation Contract

- Completion is offered only within recognized relationship sections and inserts
  resolvable references.
- Quick-fix skeletons come from `rac improve --template` / `rac schema`, matching
  the canonical template.
- "New Artifact" delegates to `rac new`; the extension never writes templates itself.

## Success Measures

- Typing in a relationship section offers ID completions; the accepted reference
  resolves (no new broken-reference diagnostic).
- A `missing-success-metrics` finding offers an "insert section" code action that
  resolves it.
- "RAC: New Artifact" creates a valid artifact via `rac new`.

## Risks

- Completion latency from spawning `rac find` per keystroke. Mitigated by the
  cached index (v0.21.6) and debouncing.
- Quick-fix skeletons drifting from templates. Mitigated by sourcing them from
  `rac improve`/`rac schema` rather than hardcoding.

## Assumptions

- `rac find` / the index, `rac improve --template`, `rac schema`, and `rac new`
  expose enough to power completion, skeletons, and scaffolding.
