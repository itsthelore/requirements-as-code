# RAC v0.26.4 — Editor Flow

## Status

Abandoned

## Context

This item is retired without separate work: when the tui-revamp series was
sketched it was scoped from a stale reading of the Explorer follow-ups, but both
pieces it planned had already shipped in the v0.8.x series and are covered by
passing tests. `rac explorer` already opens an artifact's source in an external
editor — resolving the command from the `editor` preference, then `$VISUAL` /
`$EDITOR`. Terminal editors (vim, neovim, `emacs -nw`) already suspend the
Explorer through `App.suspend()`, pause the live watcher, and rescan on resume
(`launch_editor`, `is_terminal_editor`); GUI editors keep the fire-and-forget
`Popen` path. The first-run editor prompt already runs between the welcome and
the summary (`#firstrun-editor`): Enter accepts the `$VISUAL` / `$EDITOR`
fallback, typing sets the preference, Esc skips. The Explorer still authors
nothing itself; editing belongs to the external tool (ADR-024).

The tui-revamp series therefore closes at v0.26.3. The outcomes and initiatives
below are preserved as the original record of what was planned; they describe
behaviour that already exists rather than work still to do.

## Outcomes

- **Terminal-editor support** — opening an artifact in a terminal editor
  suspends the Explorer, hands the terminal to the editor, and resumes on exit
  with the corpus rescanned, so vim / neovim users edit in place without a
  detached process.
- **A first-run editor prompt** — first run offers to set the `editor`
  preference (Enter accepts the `$VISUAL` / `$EDITOR` fallback, typing sets it,
  Esc skips), so the choice is made once and remembered.

## Initiatives

### Tier 1

- **WS1 — Suspend/resume for terminal editors** `[user-facing]`. Detect terminal
  editors and route them through `App.suspend()`, pausing the live watcher while
  suspended and rescanning on resume; GUI editors keep the fire-and-forget path.

### Tier 2

- **WS2 — First-run editor prompt** `[user-facing]`. The optional editor step
  after the welcome (already designed in `explorer-followups`), writing the
  `editor` preference; never blocking, Esc skips.

### Tier 3 — deferred

- Editor-type autodetection beyond a known terminal-editor list stays minimal;
  the preference is authoritative.

## Success Measures

- Opening an artifact in vim suspends the Explorer, returns to a rescanned corpus
  on exit, and never leaves a detached terminal process.
- First run offers the editor choice once and persists it; skipping keeps the
  `$VISUAL` / `$EDITOR` fallback.

## Risks

- `App.suspend()` interacts with the live-reload watcher. Mitigation: the watcher
  already exposes pause/resume hooks (`pause_watching` / `resume_watching`);
  reuse them around suspend.

## Assumptions

- The fire-and-forget GUI path, the `editor` preference, and the watcher
  pause/resume hooks already exist; this release adds the terminal-editor branch
  and the first-run prompt.
