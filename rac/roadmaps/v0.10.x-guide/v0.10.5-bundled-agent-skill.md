# v0.10.5 — Bundled Agent Skill

## Outcomes

- A host project can obtain the `rac-artifacts` Claude Code skill from
  the installed package alone — no copy from this repository, no
  network access, no AI involvement.
- The skill content has one canonical source. The repository's dogfood
  copy and the packaged copy cannot drift.
- The command surface for skill installation follows the existing CLI
  contracts: human output, `--json` output, exit codes, and
  never-overwrite semantics.

## Initiatives

- Ship the skill as a package resource under a new `rac.skills`
  subpackage, mirroring how canonical templates ship under
  `rac.templates` (v0.7.10, ADR-021), loaded with
  `importlib.resources`.
- Add `rac skill install [--dir PATH]`: writes
  `.claude/skills/rac-artifacts/SKILL.md` into the target project
  (default: current directory), creating parent directories, refusing
  to overwrite an existing file. Exit codes follow the standard
  convention (0 installed, 1 refused or operational error, 2 bad
  path). `--json` emits a stable contract pinned by golden tests.
  A missing packaged resource is a broken installation, reported via
  the same operational-error path as missing templates.
- Keep `.claude/skills/rac-artifacts/SKILL.md` in this repository as
  the dogfood instance; a test asserts byte-equality with the packaged
  resource.
- No skill-name argument while exactly one skill is bundled; the
  argument is introduced only when a second skill exists.

## Success Measures

- From an installed wheel in a clean environment, `rac skill install`
  produces a skill file identical to the repository copy, and Claude
  Code discovers it at the project path.
- Running the command twice exits 1 on the second run and leaves the
  existing file untouched.
- `pytest`, `rac validate rac/`, and `rac relationships rac/
  --validate` pass; golden tests pin the human and JSON output.

## Assumptions

- Project-level `.claude/skills/<name>/SKILL.md` remains the
  documented Claude Code discovery path.
- One bundled skill was sufficient when this item was drafted; the
   review and ingest skills extend it within the same release
   (v0.10.5-review-and-ingest-skills). Originally numbered v0.10.4;
   renumbered when that release identity went to Guide telemetry.

## Risks

- The Claude Code skill format or discovery path changes upstream; the
  packaged copy then needs a content release, which the sync test makes
  visible but cannot prevent.
- A second distribution surface for the same content (repo copy and
  wheel copy) invites divergence if the sync test is ever removed.
