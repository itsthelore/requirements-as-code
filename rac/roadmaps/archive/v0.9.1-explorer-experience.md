# RAC v0.9.1 — Workspace Experience

## Problem

Explorer provides repository navigation but does not yet feel like a persistent workspace.

Modern developer tools reduce friction by remembering context and guiding first-time users.

---

## Outcomes

Make Explorer feel like entering a product knowledge environment.

---

## Initiatives

## Initiative 1 — First Run Experience

Provide onboarding.

Example:

```text
Welcome to RAC Explorer

Choose Repository

> Current directory
  Recent Repository
  Browse
```

---

## Initiative 2 — Repository Analysis Flow

When opening repositories:

```text
Analyzing...

✓ Requirements
✓ Decisions
✓ Relationships

Press / for anything
```

---

## Initiative 3 — Recent Repositories

Remember previously opened repositories.

---

## Initiative 4 — Workspace Persistence

Restore:

- Last repository
- Last artifact
- Last view

---

## Initiative 5 — Preferences

Support:

- Theme
- Startup behavior
- Default views

Preference options shall follow Explorer design artifacts.

---

## Requirements

### REQ-001 — Low Friction Startup

Explorer shall be usable without prior RAC knowledge.

---

### REQ-002 — Workspace Continuity

Users shall resume previous context.

---

## Success Measures

- First-time users can open a repository and begin working without prior RAC knowledge.
- Returning users resume their previous repository, artifact, and view.
