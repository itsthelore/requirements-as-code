# RAC v0.8.1 — Explorer Navigation

## Status

Planned

## Context

v0.8.0 establishes the Explorer shell, adapter layer, and repository model.

The Explorer requirement's first capability is Discover: understand what
product knowledge exists within a repository. Users need to browse artifacts,
inspect them in context, and reach anything in the repository through a single
interaction model — without prior knowledge of RAC commands.

## Outcomes

- Users can browse, open, and inspect every recognized artifact in a
  repository from within Explorer.
- A single command surface (`/`) is the universal entry point for search and
  commands.
- First-time users reach a working repository view in seconds; returning
  users skip onboarding.

## Initiatives

### Initiative 1 — Repository Home

Provide a landing experience summarizing the repository:

```text
RAC Explorer

Health 92%

Attention

2 broken relationships
1 incomplete artifact
```

Health and attention values are rendered from Core services; scoring detail
is delivered by v0.8.2.

### Initiative 2 — Artifact Browser

Support browsing all artifact types:

- Requirements
- Decisions
- Roadmaps
- Prompts
- Designs

Artifacts default to discovery under the `rac/` root while supporting
explicitly supplied paths (ADR-018).

### Initiative 3 — Artifact Context View

Display, for any artifact:

- Metadata and identity
- Validation state
- Completeness
- Relationships
- Diagnostics

### Initiative 4 — Slash Command Surface

Introduce `/` as the universal interaction model, backed by a single command
registry (DESIGN-command-surface):

```text
/open req-001

/find payments

/health
```

- Search and command execution share one entry point; users do not
  distinguish between them.
- Artifact lookup resolves canonical IDs, legacy aliases, and keywords with
  the same semantics as `rac resolve` and `rac find` (ADR-026).
- Commands not present in the registry are not discoverable.
- Empty states teach by example.

### Initiative 5 — Keyboard Model

```text
/      command surface
↑ ↓    navigate
Enter  select
Esc    back
q      quit
```

All functionality works without mouse input.

### Initiative 6 — First Run Experience

Implement onboarding per DESIGN-first-run-experience:

- Existing RAC repository: show artifact and relationship counts, then enter
  the repository.
- Empty repository: offer import, create, or browse options.
- Invalid repository: show issue counts with the option to open anyway or
  view issues.
- Returning users skip onboarding.

Startup reaches repository context in seconds, with no login, cloud, sync, or
forced setup.

## Non-Goals

- Health scoring detail (v0.8.2)
- Recommendations (v0.8.3)
- Editing or authoring artifacts (ADR-024)
- Workspace persistence and preferences (v0.8.6)

## Success Measures

- A first-time user can open a repository and locate a named artifact without
  prior RAC knowledge.
- Every artifact reachable through the browser is also reachable through `/`.
- Everything visible in Explorer navigation is obtainable through existing
  `rac` commands and JSON output (ADR-015).
