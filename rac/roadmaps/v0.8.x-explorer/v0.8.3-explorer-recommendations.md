# RAC v0.8.3 — Explorer Recommendations

## Status

Planned

## Context

v0.8.2 makes repository health visible. Findings alone are not enough: the
Explorer requirement states that Explorer shall not merely identify issues —
it shall help users move from findings to action.

RAC Core already produces improvement guidance through the improve and review
services. Explorer's responsibility is to present that guidance so users
understand what a finding means, why it matters, and what to do about it
(DESIGN-recommendations).

## Outcomes

- Repository findings are presented with explanation, impact, and a suggested
  action.
- Users can prioritize attention using severity supplied by Core.
- Recommendations remain advisory; Explorer never applies changes.

## Initiatives

### Initiative 1 — Recommendation Categories

Present recommendations across the categories defined by
DESIGN-recommendations:

- Validation: missing sections, invalid metadata
- Relationships: broken references, missing links
- Repository health: orphaned artifacts, low traceability
- Quality: missing success measures, weak acceptance criteria

### Initiative 2 — Recommendation Presentation

Each recommendation presents:

- the finding
- why it matters (impact)
- a suggested action
- navigation to the affected artifacts

Explain before suggesting: users should understand a finding before being
asked to act on it.

### Initiative 3 — Severity

Present severity levels supplied by RAC Core:

- Critical
- Warning
- Suggestion

Severity definitions are maintained by Core intelligence, not by Explorer.

### Initiative 4 — Recommendation Navigation

From any recommendation, users can open the affected artifact's context view
and its related artifacts. Acting on recommendations (editor launch, export)
is delivered by v0.8.4.

## Non-Goals

- Automatic changes or artifact rewriting
- Replacement of human review
- Recommendation logic in Explorer (ADR-015)
- AI-dependent behaviour: recommendations remain available without AI
  (ADR-002)

## Success Measures

- A user can review repository recommendations, understand their impact, and
  locate every affected artifact without leaving Explorer.
- Every recommendation shown in Explorer is obtainable through existing `rac`
  commands and JSON output.
