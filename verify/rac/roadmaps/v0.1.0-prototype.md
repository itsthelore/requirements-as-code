# lore-verify v0.1.0 — Prototype

## Outcomes

- A walking skeleton of `lore-verify` in the `verify/` subproject: Drive, Compile,
  and Run exist end to end on a single sample target, with the runner behind the
  LV-ADR-002 interface (local implementation only).
- A compiled test passes the fidelity gate (re-run N times, green and stable)
  before acceptance, proving the `faithful-session-to-test` requirement in the
  smallest real case.
- The contract seam works one full loop: read an unverified capability from `rac
  export --graph`, verify it, and open a PR proposing a `## Verified By` line
  against a sample Lore corpus.
- The subproject is self-contained — its own packaging, tests, CI, and this
  corpus — and imports no RAC internals.

## Initiatives

### Initiative 1 — Walking skeleton

Stand up Drive (browser + terminal agent loop, BYO credentials), Compile (emit one
Playwright test with intent-level assertions), and Run (local runner behind the
interface) against one sample app, on the host OS only.

### Initiative 2 — Fidelity gate

Implement the N-run stability check in Compile and make acceptance depend on it
(`faithful-session-to-test` REQ-002). Reject flaky output rather than commit it.

### Initiative 3 — Contract loop

Read the verification worklist from `rac export --graph` and open a proposal PR
adding `## Verified By` lines, closing the loop with the RAC-side
`capability-verification-coverage` (RAC ADR-065, ADR-063).

## Success Measures

- One sample capability goes from "unverified in `rac coverage`" to a merged
  `## Verified By` reference via a `lore-verify`-opened PR.
- The one compiled test is green and stable across N headless re-runs.
- `verify/`'s suite and lint pass with no `rac` source on the path.

## Assumptions

- A single sample app and a sample Lore corpus are enough to exercise the full
  loop; breadth (multi-OS, multi-target, the hosted runner) comes after the
  prototype.
- The RAC-side `verified-by` export edge (`capability-verification-coverage`) is
  available, or stubbed against its documented shape until it ships.

## Risks

- The prototype over-scopes into breadth before the loop closes. Mitigation: one
  target, one OS, one capability — depth over breadth for v0.1.0.
- Intent extraction in Compile proves harder than expected. Mitigation: it is the
  first thing prototyped (the design's lead Open Question).
